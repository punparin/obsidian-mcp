Glossary

cos_simcosine similarity

How close two embedding vectors point in 384-dimensional space. 0 = unrelated, 1 = identical direction. This is the raw semantic match before any graph signal is mixed in.

cos_sim = 1 − cosine_distance
embeddingBGE-small, 384-dim

A vector representation of text where similar meaning lands at similar vectors. We use BAAI/bge-small-en-v1.5, a local ONNX model (~100MB) — runs on CPU with no API call. Queries and notes share the same vector space, so cosine similarity is meaningful between any pair.

chunkmarkdown-aware slice

A piece of a note we embed independently — typically an H2 or H3 section, packed up to 1600 chars with paragraph overlap. Long notes split into many chunks; tiny sections merge with neighbors. We embed chunks (not whole notes) so retrieval lands on the right paragraph.

kNN / ktop-k retrieval

k-nearest neighbors: the top k chunks closest to the query vector by cosine distance, fetched from sqlite-vec. The k input above sets how many to pull before the graph re-rank narrows down. Internally we kNN at k=50 chunks then aggregate to notes.

wikilink[[note-name]]

An Obsidian-style link from one note to another, written [[target-note]]. If your query body contains one, the linked note gets a fixed +1.0 wikilink_match boost — your hand-curated graph beats marginal semantic noise.

tag_jaccardtag overlap

How much the query's #tags overlap with the note's tags, measured by Jaccard similarity (set intersection over set union). 1.0 = identical tag set, 0 = disjoint. Catches notes that share the same topic taxonomy even when wording differs.

jaccard(A, B) = |A ∩ B| / |A ∪ B|
neighbor_hopsgraph distance

Distance through the wikilink graph from a note explicitly mentioned in your query. 1-hop = directly linked, 2-hop = linked-by-something-linked. We BFS up to 2 hops; further notes get no bonus.

neighbor_bonus = 1 / hops (1.0 at 1-hop, 0.5 at 2-hop)
recency180-day half-life

Exponential decay based on the note's modified date — fresher notes get a small lift without dropping older foundational notes off a cliff. Half-life of 180 days means a 6-month-old note gets ~0.5; year-old gets ~0.25.

recency = exp(−age_days / 180)
contributionweighted signal

What each signal actually added to the final score after the slider's weight was applied. The horizontal bars on each result card show contributions side-by-side, so you can see whether a note ranked high because of cosine match, wikilinks, tags, or recency.

contribution = weight × raw_signal
scorefinal ranking

The sum of all five weighted contributions. This is what sorts the result list, top-down. Notes with similar cosine similarity can land in very different positions once the graph signals are layered in.

final = w_sem · cos_sim + w_link · wikilink_match + w_tag · tag_jaccard + w_neighbor · neighbor_bonus + w_recency · recency
MCPModel Context Protocol

The protocol Claude Code uses to talk to external tools (this server, GitHub, Slack, etc.). Servers expose tools and resources over a STDIO or HTTP transport. The Vault Explorer is a sidecar — separate HTTP server, same vault index — so the MCP transport stays narrow.
